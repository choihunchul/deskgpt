# Security Policy

## Supported Versions

DeskGPT security fixes are published for the latest release branch and the current `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Use GitHub Security Advisories to report the issue privately:

1. Go to the repository's Security tab.
2. Choose **Report a vulnerability**.
3. Include as much of the following as you can:
   - A short description of the issue
   - Affected DeskGPT version(s)
   - Steps to reproduce
   - Expected and actual behavior
   - Proof of concept, screenshots, or logs if available

## What to Include

Helpful reports usually include:

- A clear impact statement
- Whether the issue is local-only or affects network traffic, authentication, or update delivery
- Any user action required to trigger the issue
- A minimal reproduction path

## Response Expectations

We will review reports as soon as practical and coordinate a fix or mitigation before public disclosure when possible.

## Safe Handling

If you are unsure whether something is a vulnerability, err on the side of reporting it privately rather than posting it publicly.
